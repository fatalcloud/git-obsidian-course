### Ключевые идеи раздела: Собираем пазл

Давайте кратко вспомним, что именно мы заложили в фундамент ваших знаний:

1.  **Зачем нужна VCS (Система контроля версий):** Это не просто «сохранение». Это страховка от ошибок, **машина времени** и инструмент для командной работы без хаоса. Git, в отличие от старых централизованных систем, даёт вам **полную копию истории** на локальный диск, делая вас независимым от сети и единого сервера.

2.  **Установка и первичная настройка:** Мы научились устанавливать Git на любой ОС и, что важнее, **представились** ему командами `git config --global user.name` и `user.email`. Это ваше цифровое имя, которое будет стоять под каждым сохранённым изменением. Мы разобрались, что настройки бывают системные, глобальные и локальные.

3.  **Рождение проекта (`git init` vs `git clone`):** Мы поняли разницу между созданием проекта с нуля в своей папке (`init`) и получением копии существующего проекта из интернета (`clone`). Вы теперь знаете, что скрывается в таинственной папке **`.git`** — это и есть сам репозиторий, «чёрный ящик» с историей.

4.  **Три состояния и `git status`:** Мы запомнили, что файл может быть **изменённым** (рабочий каталог), **подготовленным** (индекс) и **зафиксированным** (репозиторий). Команда `git status` — ваш главный навигатор, который всегда подскажет, на каком вы свете.

Увидишь надпись на английском — значит, полёт нормальный. Теперь ты готов сохранять историю изменений, завтра покажу как".*

**Ключи к вопросам для самопроверки:**
1.  **Да, могу.** Потому что Git — распределённая система. Вся история хранится в папке `.git` на вашем ноутбуке (локальный репозиторий). Интернет нужен только для синхронизации с удалённым репозиторием (GitHub), но не для локальной работы.
2.  Нужно перейти в папку проекта и выполнить: `git config --local user.email "work@company.com"`. Флаг `--local` запишет настройку в `.git/config` внутри этого проекта. Она переопределит глобальную. Если этого не сделать, все коммиты пойдут с домашней почты, что может быть нарушением политики компании.
3.  **Если коммита не было** — восстановить из Git **нельзя**. Git видит только то, что попало в его базу (в `.git`). **Если коммит был** — **можно**. "Спасённая" версия хранится в виде сжатого объекта внутри папки `.git`. Git по команде извлечёт её обратно в рабочий каталог.
4.  **Ничего не сломали и клонировать заново не нужно.** Git просто проверил наличие папки `.git`, обновил шаблоны хуков (если они были изменены) и сообщил вам об этом. История коммитов и связь с сервером остались нетронутыми. Просто не делайте так без необходимости, это бесполезное действие.
