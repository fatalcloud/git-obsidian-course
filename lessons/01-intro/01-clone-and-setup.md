# Урок 1.1: Клонирование репозитория курса и настройка окружения

## Для кого этот урок
- **Студенты курса** – вы будете только читать и выполнять задания локально.
- **Энтузиасты** – кто хочет дополнить курс, читайте `CONTRIBUTING.md` после этого урока.

## Цель
Научиться клонировать удалённый репозиторий, настраивать Git и выполнять задания в своей локальной копии.

## Инструкция для студента

1. **Клонируйте репозиторий курса**  
   ```bash
   git clone https://github.com/ваш_логин/git-mastery-course.git
   cd git-mastery-course
   ```

2. **Настройте ваше имя и email** (если ещё не сделали глобально)  
   ```bash
   git config user.name "Ваше Имя"
   git config user.email "your@email.com"
   ```
   > [!note] Это нужно только для локальных коммитов, если вы будете их делать в своих учебных ветках. Курс не требует от вас пушить обратно.

3. **Проверьте статус**  
   ```bash
   git status
   ```
   Вы должны увидеть «nothing to commit, working tree clean».

4. **Создайте свою локальную ветку для выполнения заданий** (не будете пушить)  
   ```bash
   git checkout -b my-solutions
   ```

5. **Выполните задание**  
   Создайте в корне репозитория файл `my-name.txt` со своим именем и сделайте локальный коммит:  
   ```bash
   echo "Меня зовут [ваше имя]" > my-name.txt
   git add my-name.txt
   git commit -m "feat(student): add my name locally"
   ```

6. **Убедитесь, что коммит остался локально**  
   ```bash
   git log --oneline
   ```
   Вы увидите свой коммит. Он **не** отправится на сервер, потому что вы не выполняли `git push`.

## Результат
Вы склонировали курс, настроили Git, создали локальную ветку и сделали коммит. Всё это осталось на вашем компьютере.

## Для энтузиастов (как внести изменения в курс)
Если вы хотите исправить опечатку, добавить урок или перевести курс:
1. Сделайте **форк** этого репозитория на GitHub.
2. Клонируйте **свой форк**.
3. Создайте ветку по правилам (например, `fix/typo-lesson1` или `translate/ru`).
4. Внесите изменения, сделайте коммит (следуя [Conventional Commits](https://www.conventionalcommits.org/)).
5. Запушьте ветку в свой форк и откройте **Pull Request** в оригинальный репозиторий.
6. Дождитесь ревью.

Подробные правила — в файле `CONTRIBUTING.md` (появится позже).
