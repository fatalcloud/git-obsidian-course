# Правила внесения изменений в курс

Спасибо, что хотите улучшить курс! Мы придерживаемся профессионального Git-воркфлоу.

## Процесс

1. **Форк** репозитория на GitHub.
2. **Клонируйте свой форк** локально.
3. **Создайте ветку** с понятным именем:
   - `fix/краткое-описание` – для исправлений ошибок.
   - `feat/название-урока` – для новых уроков.
   - `translate/ru` (или `es`, `fr`) – для переводов.
4. **Вносите изменения**.
5. **Коммиты** – по [Conventional Commits](https://www.conventionalcommits.org/):
   - `fix(lesson1): исправить опечатку`
   - `feat(lesson5): добавить раздел о git bisect`
   - `docs(readme): обновить установку`
6. **Запушьте ветку** в свой форк.
7. **Откройте Pull Request** в оригинальный репозиторий (`main` ветку).
8. **Дождитесь ревью**. Возможны правки.

## Требования к содержимому

- Все уроки в формате Markdown (`.md`).
- Используйте подзаголовки `##`, `###`.
- Команды Git оформляйте в ```bash ... ```.
- Добавляйте карточки для запоминания в конце каждого урока (блок `## Карточки` с `> [!question]`).
- Если описываете процесс ветвления, добавляйте Mermaid-диаграмму (см. примеры).
- Для переводов: создавайте папку `locale/ru/` и копируйте туда структуру уроков, не изменяя оригинал.

## Автоматические проверки

При PR запускаются:
- Markdown lint (проверка форматирования)
- Prettier (для единого стиля)
- (Опционально) проверка ссылок

Убедитесь, что ваши изменения проходят проверки.
